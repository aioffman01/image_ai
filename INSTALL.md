# 💻 Windows 배포형 애플리케이션 설치 및 실행 가이드

본 가이드는 `setup.win.d` 디렉터리에 빌드된 Windows용 독립 실행형(배포형) 이미지 AI 분석 프로그램의 설치 및 실행 방법을 설명합니다.

---

## 📁 배포 패키지 구성 요소 (`/setup.win.d`)

* **`local_ocr_app.exe`**: 메인 실행 파일 (더블 클릭하여 실행)
* **`flutter_windows.dll`**: Flutter 엔진 라이브러리
* **`platform_ocr.dll`**: Windows 네이티브 OCR 연동 플러그인
* **`file_selector_windows_plugin.dll`**: 로컬 파일 탐색기 연동 플러그인
* **`data/`**: 앱 실행에 필요한 리소스 및 폰트 에셋 폴더

---

## 🚀 실행 및 사용 방법

### 1단계: 실행 파일 구동
1. 워크스페이스 내의 `setup.win.d` 폴더로 이동합니다.
2. **`local_ocr_app.exe`** 파일을 더블 클릭하여 실행합니다.
   * *참고: 본 앱은 포터블(Portable) 버전으로, 별도의 인스톨러 설치 과정 없이 압축을 풀거나 폴더 채로 다른 PC에 이동하여 즉시 실행할 수 있습니다.*

### 2단계: 이미지 분석 및 OCR 진행
1. 앱 좌측의 사이드 메뉴에서 **이미지 등록** 메뉴를 선택합니다.
2. **이미지 불러오기** 버튼을 눌러 분석하고자 하는 로컬 사진 파일을 탐색기에서 선택합니다.
3. 사진이 화면에 로드되면 **분석 실행 (AI)** 버튼을 클릭합니다.
4. 프론트엔드 자체에 내장된 로컬 OCR 엔진이 구동되어 사진 속 텍스트가 추출되고, 추출된 텍스트와 신뢰도(100% 로컬)가 화면에 출력됩니다.

### 3단계: 촬영 정보 및 지도 확인
* 선택한 이미지에 촬영 날짜 및 GPS 위치 정보가 포함되어 있는 경우, 결과 카드 영역에 위경도 좌표 및 촬영 시각이 표시되며 **OpenStreetMap 기반의 고해상도 지도**가 300px 높이로 자동 렌더링되어 핀 포인트를 표시합니다.
* 위치 정보가 없는 사진은 자동으로 지도가 노출되지 않고 날짜 정보만 식별하여 표시합니다.

---

## ⚠️ 중요 참고 사항

* **완전한 오프라인/로컬 독립 실행**: 
  * 본 프로그램은 프론트엔드 내부에서 Windows OS의 내장 OCR 엔진(`Windows.Media.Ocr`)을 활용해 동작하므로, **백엔드 서버 구동이나 인터넷 연결이 없어도 정상적으로 OCR 분석이 동작**합니다.
  * 따라서 배포 시 백엔드 서버를 함께 배포하지 않고, 이 `setup.win.d` 폴더만 배포해도 안전하게 사용하실 수 있습니다.

## 🛠️ 스마트 앱 컨트롤(Smart App Control) 차단 해결 방법

Windows 11의 **스마트 앱 컨트롤(SAC)** 기능은 디지털 서명(인증서)이 없는 임의의 컴파일된 실행 파일(`*.exe`)의 구동을 기본적으로 차단합니다. 이를 해결하기 위한 주요 방법들을 설명합니다.

### 방법 1: 파일 차단 해제 (속성 변경)
1. `setup.win.d/local_ocr_app.exe` 파일을 마우스 우클릭한 후 **속성(Properties)**을 클릭합니다.
2. **일반(General)** 탭 가장 하단에 있는 **보안(Security)** 섹션을 확인합니다.
3. **"차단 해제(Unblock)"** 체크박스에 체크한 뒤, **적용** 및 **확인**을 누릅니다.

### 방법 2: Windows 보안에서 제외(Exclusion) 등록
1. Windows 검색창에 **"Windows 보안(Windows Security)"**을 입력하여 앱을 엽니다.
2. **바이러스 및 위협 방지** -> **설정 관리**를 클릭합니다.
3. 하단의 **제외** 항목에서 **제외 추가 또는 제거**를 선택합니다.
4. **제외 추가** 버튼을 누르고 **폴더**를 선택한 뒤, 빌드된 폴더인 `setup.win.d`의 절대 경로를 지정하여 추가합니다.

### 방법 3: 로컬 인증서 자가 서명 (Self-Signing)
개발자 환경에서 영구적으로 경고를 없애기 위해, 로컬에서 서명 인증서를 발급하여 서명할 수 있습니다.
1. PowerShell을 **관리자 권한**으로 엽니다.
2. 아래 명령어를 입력하여 로컬 서명 인증서를 생성합니다:
   ```powershell
   $cert = New-SelfSignedCertificate -Type CodeSigningCert -Subject "CN=LocalOcrApp" -CertStoreLocation "Cert:\CurrentUser\My"
   ```
3. 생성한 인증서를 신뢰할 수 있는 루트 기관에 등록합니다:
   ```powershell
   Move-Item -Path $cert.PSPath -Destination "Cert:\CurrentUser\Root"
   ```
4. 실행 파일에 서명 처리를 수행합니다:
   ```powershell
    Set-AuthenticodeSignature -FilePath "C:\Users\redof\Documents\Antigravity\ImageAI\setup.win.d\local_ocr_app.exe" -Certificate $cert
    ```

## 👥 타인에게 배포 시 유의 사항

본 프로그램을 다른 사용자 또는 외부 컴퓨터에 배포하고자 할 때는 아래 사항을 준수 및 안내해야 합니다.

### 1. 보안 경고 안내 (스마트스크린 / 스마트 앱 컨트롤)
* **현상**: 로컬 자가 서명(Self-Sign)은 **최초 서명을 진행한 PC에서만 신뢰**됩니다. 따라서 타인 PC에 폴더 채로 배포할 시 여전히 차단 경고창이 발생할 수 있습니다.
* **해결 안내**: 배포 대상자에게 본 설명서의 **[방법 1: 파일 차단 해제]**를 수행한 뒤 실행하도록 반드시 안내해 주십시오. 
* **대외 상용 배포 시**: 경고 창을 완전히 없애기 위해서는 공인 인증기관(Sectigo, DigiCert 등)에서 유료 **코드 서명 인증서(Code Signing Certificate)**를 별도 구매하여 최종 서명해야 합니다.

### 2. 구동 요구사항 (OS 환경)
* 본 앱의 핵심 OCR 기능은 Windows OS 내부 엔진(`Windows.Media.Ocr`)을 활용합니다. 따라서 배포 대상자의 운영체제가 **Windows 10 또는 Windows 11 이상**이어야 분석 기능이 원활히 동작합니다.

### 3. 지도 API 사용량 가이드
* 현재 지도 창은 오픈스트리트맵(OpenStreetMap) 무료 서버의 타일 이미지를 실시간 요청하여 렌더링하고 있습니다.
* 소규모 팀 및 사내 공유는 무관하지만, **대규모 대외 배포 및 상용화**를 계획하시는 경우 OSM 사용 조건(Tile Usage Policy)에 위배될 수 있으므로, 상용 지도 서비스(Mapbox, Google Maps 등)의 API 키 연동 방식으로 전환해야 합니다.
